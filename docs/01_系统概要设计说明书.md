### 1.2 与原 Java 项目架构对比

| 层次 | 原项目（Java/Spring） | 本项目（Go/Gin） |
|------|----------------------|-----------------|
| Web 层 | Spring MVC + DispatcherServlet | Gin 路由 + 中间件链 |
| 安全层 | Spring Security + Filter Chain | 自定义 JWT 中间件 |
| ORM 层 | Spring Data JPA + Hibernate | GORM（类似 JPA but Go 原生） |
| 事务 | @Transactional 声明式 | 手动物务（db.Transaction） |
| 缓存 | Spring Cache + Redis | Redis（按需集成） |
| 消息 | RocketMQ（异步解耦） | 暂未迁移（可按需添加） |
| 异步任务 | @Async + ThreadPool | goroutine + channel |

## 3. 核心业务流

### 3.1 用户认证流程

```
用户提交 username + password
         ↓
  POST /api/auth/login
         ↓
   bcrypt 验证密码哈希
         ↓
  生成 JWT Access Token（24h）
  生成 Refresh Token（7d）
         ↓
  返回 {token, refreshToken, userInfo}
         ↓
  前端在 Header 携带：Authorization: Bearer <token>
```

### 3.2 订单创建流程

```
POST /api/order/orders
         ↓
 校验请求参数（senderName/phone/address 必填）
         ↓
 生成订单号 ORD+时间戳+随机数
         ↓
 INSERT o_order → 返回主键
         ↓
 批量 INSERT o_order_item（明细）
         ↓
 写入 o_order_status_log（初始状态：待确认 10）
         ↓
 返回完整订单对象（含 statusName）
```

### 3.3 订单状态流转

```
10(待确认) → 20(已确认) → 30(已入库)
    ↓            ↓            ↓
  80(取消)    80(取消)     80(取消)
                              ↓
                         40(已发货)
                              ↓
                         50(运输中)
                              ↓
                         60(已送达)
                              ↓
                         70(已完成)
```

| 对象 | 规范 | 示例 |

## 6. 安全性设计

### 6.1 JWT 认证流程

```
请求携带 Header：
  Authorization: Bearer <token>
         ↓
  JWTAuth 中间件拦截
         ↓
  提取 Bearer token → ParseToken()
         ↓
  验证签名 + 检查过期
         ↓
  解析出 username/roleCode → 写入 Context
         ↓
  后续 handler 通过 c.Get("username") 获取
```

### 6.2 角色权限控制

```go
// 路由级别
auth := r.Group("/api", middleware.JWTAuth())

// 或方法级别
auth.GET("/admin-only", middleware.RoleCheck("ADMIN", "SUPER_ADMIN"))
```

### 6.3 CORS 跨域

所有 origins 允许（`*`），支持 GET/POST/PUT/DELETE/OPTIONS，支持 Authorization/Refresh-Token 头。

## 8. 部署架构

### 8.1 单二进制部署

```bash
# 编译
go build -o logistics-gin .

# 运行（无需 JVM 或任何运行时）
./logistics-gin

# 或指定端口
PORT=8081 ./logistics-gin
```

### 8.2 Docker 化（可选）

```dockerfile
FROM ubuntu:24.04
COPY logistics-gin /app/logistics-gin
CMD ["/app/logistics-gin"]
```

容器镜像预计 < 30MB（相比 JAR + JVM 方案 200MB+）。

### 8.3 资源占用对比

| 指标 | 原项目（Java JAR） | 本项目（Go 二进制） |
|------|------------------|-------------------|
| 启动后内存 | ≥768MB | ~30MB |
| 二进制大小 | JAR ~50MB + JVM ~150MB | ~19MB |
| 冷启动时间 | ~10-15s | ~50ms |
| 并发模型 | 线程（线程池限制） | 协程（goroutine，10万+ 并发） |
