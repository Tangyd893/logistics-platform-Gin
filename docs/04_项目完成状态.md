## 🔄 与原 logistics-platform 的 API 兼容性

**结论：前端无需任何修改即可直接切换到 Gin 后端。**

- 响应格式完全兼容：`{code, message, data, timestamp}`
- 分页格式完全兼容：`{records, total, pages, page, size}`
- 认证方式完全兼容：`Authorization: Bearer <token>`
- 接口路径完全兼容：与原 Spring Boot 接口一一对应

## 📦 部署方式

```bash
# Docker 全家桶（一键启动）
docker compose up -d

# 本地开发
cd backend && ./logistics-gin   # 后端（8080）
cd frontend && npx vite        # 前端（3000）
```

**访问：**
- 前端：`http://localhost:3000`
- 后端：`http://localhost:8080`
- 账号：`admin / admin123`
