## 1. 接口规范

所有接口统一响应格式：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... },
  "timestamp": "2026-04-20T13:00:00+08:00"
}
```

### 1.1 认证模块

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | `/api/auth/login` | 用户登录 | 否 |
| POST | `/api/auth/refresh` | 刷新令牌 | 否（Header 带 Refresh-Token） |
| GET | `/api/auth/me` | 当前用户信息 | 是 |
| POST | `/api/auth/logout` | 登出 | 是 |

#### POST /api/auth/login

**请求：**
```json
{
  "username": "admin",
  "password": "admin123"
}
```

**响应（成功）：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiJ9...",
    "expiresIn": 86400,
    "userInfo": {
      "username": "admin",
      "displayName": "系统管理员",
      "roleCode": "ADMIN"
    }
  }
}
```

#### GET /api/auth/me

**响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "username": "admin",
    "displayName": "系统管理员",
```

#### PUT /api/order/orders/{id}/status

**请求：**
```json
{
  "status": 20,
  "remark": "已确认备货"
}
```

**状态码说明：**

| 值 | 名称 | 说明 |
|----|------|------|
| 10 | 待确认 | 订单创建初始状态 |
| 20 | 已确认 | 客服确认订单 |
| 30 | 已入库 | 仓库已收货入库 |
| 40 | 已发货 | 已出库发货 |
| 50 | 运输中 | 配送途中 |
| 60 | 已送达 | 已完成配送 |
| 70 | 已完成 | 客户确认收货 |
| 80 | 已取消 | 订单取消 |

**状态流转约束：**
- 10→20/80、20→30/80、30→40/80、40→50、50→60/70、60→70

## 2. 认证说明

除 `/api/auth/login`、`/api/auth/refresh`、`/health` 外，所有接口均需要 JWT 认证。

**请求 Header：**
```
Authorization: Bearer <token>
```

**认证失败响应（HTTP 401）：**
```json
{
  "code": 401,
  "message": "令牌无效或已过期",
  "timestamp": "2026-04-20T13:00:00+08:00"
}
```

## 3. 错误码对照

| HTTP 状态码 | 业务 code | 说明 |
|-------------|-----------|------|
| 200 | 200 | 成功 |
| 400 | 400 | 请求参数错误 |
| 401 | 401 | 未认证 / 认证失败 |
| 403 | 403 | 无权限访问 |
| 404 | 404 | 资源不存在 |
| 500 | 500 | 服务器内部错误 |
