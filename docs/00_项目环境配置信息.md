## 4. 启动方式

### 4.1 基础设施（PostgreSQL）

```bash
cd ~/.openclaw/workspace/output/logistics-platform/docker
docker compose up -d postgres

# 初始化表结构（如尚未初始化）
docker exec -i logistics_postgres psql -U logistics_user -d logistics \
  < ~/.openclaw/workspace/output/logistics-platform-Gin/sql/init.sql
```

### 4.2 启动 Gin 服务

```bash
cd ~/.openclaw/workspace/output/logistics-platform-Gin
./logistics-gin
# 或手动编译后运行
go build -o logistics-gin . && ./logistics-gin
```

服务地址：`http://localhost:8080`

### 4.3 环境变量（可选）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `8080` | 服务端口 |
| `DB_HOST` | `localhost` | PostgreSQL 主机 |
| `DB_PORT` | `5432` | PostgreSQL 端口 |
| `DB_USER` | `logistics_user` | 数据库用户名 |
| `DB_PASSWORD` | `logistics_pass` | 数据库密码 |
| `DB_NAME` | `logistics` | 数据库名 |
| `JWT_SECRET` | `logistics-platform-secret-key-256bit` | JWT 签名密钥 |

## 9. 与原项目对比

## 10. 注意事项

### Go 工具链路径

项目使用 `/tmp/gotool/go` 下的独立 Go 1.23.4（避免与系统预装 Go 冲突）。编译/测试时需指定：

```bash
export PATH=/tmp/gotool/go/bin:$PATH
export GOPROXY=https://goproxy.cn,direct
```

### bcrypt 密码哈希

admin 用户密码哈希通过 Go 程序生成，已同步到代码（`main.go` 中的 seed）和数据库。如需修改，参考：

```bash
go run ./cmd/genpwd/main.go   # 生成新哈希
```
